# Page

In order to have a correct layout semantic and a sticky footer on all pages, I generally use a global structure like this model:

```html
<!DOCTYPE html>
<html>
  <head>
    <title>Page title</title>
    <meta charset="utf-8" />
    <meta name="description" content="Page description" />
    <meta http-equiv="X-UA-Compatible" content="IE=edge" />
    <meta name="viewport" content="width=device-width, initial-scale=1" />
  </head>
  <body class="o-page">
    <div class="o-page__container o-container">
      <header class="p-header" role="banner">
        Header
      </header>
      <main class="o-page__content" role="main">
        Content
      </main>
    </div>
    <footer class="p-footer" role="contentinfo">
      Footer
    </footer>
  </body>
</html>
```

`o-page` class has a `is-fixed` state class that will fix the scroll, to use when a pop-in is displayed for example.
